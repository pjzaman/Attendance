# Apon Attendance

HR/attendance suite for Apon Plastic, built around a ZKTeco UFace800 biometric
reader. Two cooperating components:

- **Flutter app** (Windows desktop today; Android/iOS/Web ready) — what HR,
  managers, and employees actually use. Reads/writes Firestore directly.
- **Python bridge** (`apon-bridge.exe`) — a headless service running on the
  PC that's physically wired to the device. Polls the ZKTeco, pushes punches
  to Firestore, listens for commands queued by the Flutter app.

Designed for multi-user from day one. Any HR laptop signed in to the right
Firebase project sees the same live data. The bridge is the only thing that
needs to be on the same LAN as the biometric.

## Architecture at a glance

```
   Employee touches face/finger
            ↓
   ┌──────────────────┐
   │ ZKTeco UFace800  │  (LAN, 192.168.x.x:4370)
   └─────────┬────────┘
             │ pyzk over TCP
             ▼
   ┌──────────────────────────────┐
   │ apon-bridge.exe              │  ← service-account auth
   │ on the wired PC              │     (24/7 Windows scheduled task)
   └─────────┬────────────────────┘
             │ firebase-admin SDK (HTTPS)
             ▼
   ┌──────────────────────────────┐
   │ Firestore                    │
   │   punches/{id}               │
   │   bridges/{id}   (heartbeat) │
   │   bridge_commands/{id}       │
   │   employees, shifts, …       │
   └─────────┬────────────────────┘
             │ Firestore real-time listeners (firedart)
             ▼
   apon_attendance.exe on HR's laptop, manager's phone, etc.
```

The Flutter app never opens a socket to the device. The bridge never opens
an inbound port — it only subscribes/writes to Firestore.

## What's in the repo

```
lib/                         Flutter app source
  main.dart                  Firebase init, then runApp(AponAttendanceApp)
  app.dart                   MaterialApp shell, routes to AuthGate
  firebase_options.dart      Generated by `flutterfire configure`
  shared/                    Theme + design tokens
  models/                    24 data classes (Employee, Shift, Schedule, …)
  providers/app_state.dart   Single ChangeNotifier; subscribes to ~22 Firestore
                             collections, exposes lists/maps to screens
  services/
    auth/auth_service.dart   FirebaseAuth wrapper (email + Google sign-in)
    firestore/firestore_repo.dart    All Firestore reads/writes
    export_service.dart      CSV / XLSX exporters
    *_seed.dart              First-launch defaults (shifts, leave types, etc.)
  screens/                   30+ screens: dashboard, employees (12-tab detail),
                             schedules, attendance, requests, payroll, settings
  widgets/                   Shared UI: AppShell, KpiCard, StatusPill, …

bridge/                      Python service
  pyproject.toml
  .env.example
  apon-bridge.spec           PyInstaller config
  run.py                     Entry point (PyInstaller-friendly)
  bridge/                    Package
    main.py                  Orchestrator loop
    config.py                Env loading
    device.py                pyzk wrapper + reconnect-with-backoff
    firestore_writer.py      Idempotent batch writes
    heartbeat.py             bridges/{id} updater
    command_listener.py      Firestore subscriber
    command_executor.py      Runs queued commands against the device
    state.py                 Local sqlite (high-water-mark, processed cmds)
  install/
    install.ps1              Run as Admin on target PC; registers a
                             scheduled task that auto-starts at boot
    uninstall.ps1

deploy/
  build_app.ps1              Builds the Flutter Release zip
  build_bridge.ps1           Builds apon-bridge.exe via PyInstaller
                             + assembles install bundle

firestore.rules              Deployable security rules
firebase.json                Firebase CLI config
.firebaserc                  Project mapping (attendance-app-apon)

Attendance_Program_Modification_Plan.docx    Original Rysenova-benchmarked plan
SETUP.md                                     Detailed setup + deploy guide
```

## Getting started

See [SETUP.md](SETUP.md) for the dev-laptop and target-machine workflows.
TL;DR for development:

```powershell
git clone https://github.com/pjzaman/Attendance.git
cd Attendance
flutter pub get
flutter run -d windows
```

(You'll need to run `flutterfire configure` once to generate
`lib/firebase_options.dart` against your own Firebase project — the values
checked in here point to `attendance-app-apon`.)
